Business Systems

Three realistic apps acting as live security sensors

Each domain—Banking, Airline, and Medical—is a complete application with its own users, workflows, and security telemetry, all streaming into the SentinX Security Platform.

Bank System

Full online banking with built‑in security and detailed audit trails

BUSINESS FEATURES

🏦 Digital banking experience

  • Customer lifecycle: registration, login, profile, and KYC (national ID, phone, address).
  • Accounts: saving, current, prepaid accounts with balances and statements.
  • Cards: debit/credit/prepaid requests, issuance, blocking/unblocking, and card limits.
  • Money movement: internal transfers, bill payments, deposits, and withdrawals.
  • Loans: loan requests, document uploads, schedules, and installment views.
  • Notifications: in‑app messages for approvals, alerts, and support.
SECURITY & LOGGING

🛡️ Local security sensor

  • Strong passwords, national ID validation, and optional multi‑factor authentication.
  • Tracking of logins, failures, logouts, password resets, and profile changes.
  • High‑value operations logged (large transfers, new device logins, card blocks).
  • Attack detection layer for SQL injection, XSS, path traversal, command injection, and suspicious file uploads.
  • Brute‑force detection based on failed logins per IP/email over time windows.
  • Structured security events (who, what, when, where, severity, context) pushed to the SentinX Security Dashboard.

Airline System

Modern travel booking with OAuth login and fine‑grained monitoring

BUSINESS FEATURES

✈️ Booking portal

  • User accounts with registration and classic login.
  • OAuth integration with Google and GitHub for quick sign‑in.
  • Flight search by destination, dates, and trip type.
  • End‑to‑end booking: flight → seat/class → extras → confirmation.
  • “My bookings” area, profile management, comments, and support.
  • Admin console for offers, prices, classes, users, and comment moderation.
SECURITY & LOGGING

🛰️ Telemetry‑rich application

  • Centralized input sanitization and validation to reduce XSS and injection risk.
  • Logging of user actions (search, start booking, confirm/cancel booking).
  • Auth events: login successes/failures, admin logins, password reset flows.
  • Admin operations: creating offers, changing prices, moderating content.
  • Technical events: database errors and exceptions captured for analysis.
  • Security‑perspective events such as repeated failed logins or suspicious payloads.
  • Batch or real‑time JSON logs pushed to Dashboard with system name, category, severity, IP, etc.

Medical System

Healthcare workflows with strong access control and sensitive‑data logging

BUSINESS FEATURES

🏥 Healthcare management

  • Roles for patients, doctors, and admins with dedicated dashboards.
  • Patient data including DOB, gender, blood type, and medical history.
  • Appointment scheduling and management between doctors and patients.
  • Medical records, prescriptions, and downloadable lab results or PDFs.
  • Admin tools for user management and uploading patient files.
SECURITY & LOGGING

🔒 Protected medical context

  • Dedicated session namespace so it does not clash with the Airline system.
  • Strict directory control—config, vendor, and logs are protected; uploads are non‑executable.
  • Role‑based authentication and routing for patient/doctor/admin dashboards.
  • Rate limiting and lockout after repeated failed logins from a single IP.
  • Logs for logins/logouts, profile changes, appointments, and record updates.
  • Monitoring of access to sensitive resources such as prescriptions and files.
  • Events buffered and sent to the Dashboard with contextual metadata.
