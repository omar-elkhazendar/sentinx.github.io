Security Dashboard

From raw logs to intelligent, actionable defense

The SentinX Security Dashboard is the central SOC‑style console that ingests events from all systems, runs AI classification, and orchestrates alerts and blocking.

Data Ingestion

Unified security telemetry from Bank, Airline, and Medical systems

Log & Attack APIs

Each business system acts as a security sensor that continuously pushes telemetry to the Dashboard via HTTP.

Log endpoint

General security and activity events are sent as structured JSON with fields like system, user, category, context, and severity.

Attack endpoint

High‑confidence or explicitly detected attacks (e.g. brute‑force, SQLi patterns) are reported to a dedicated API for faster triage.

Normalization

The Dashboard validates payloads, enriches them with receive time and source metadata, and maps them to a common schema.

Central store

All events are stored in a central security database: logs, attacks, predictions, system health, devices, and more.

AI Classification Pipeline

Models that understand behavior, not just messages

From event to labeled security insight

Preprocessing

The pipeline extracts important text (URLs, parameters, error messages), normalizes casing, removes noise, and masks sensitive identifiers like emails and IPs.

Multiple models

Classical ML and deep learning models are trained on logs and attack data to detect anomalies and attacks versus normal activity.

Unified categories

Model outputs are mapped to a small, consistent set of categories such as normal, brute‑force login, SQL injection attempt, XSS, suspicious file upload, or unusual access.

Risk scoring

Each event receives a confidence value and risk score, which later drive alerting, escalation, and automated blocking decisions.

Visualization & Monitoring

A mini SOC dashboard focused on the three systems

Security views

Attack dashboard

Shows the latest and active attacks across all systems with severity coloring, filters by system, and quick drill‑down into context.

AI dashboards

Per‑model views (e.g. CNN, RNN, Naive Bayes) with stats on how many events were tagged as attack vs. normal, top attack types, and confusion summaries.

General monitoring

Overall event rates, per‑system trends, recent alerts, and top source IPs to help security teams understand what is happening right now.

Profiles & settings

Security operator accounts, profile images, password changes, global thresholds (e.g. auto‑block risk score), and configuration export/import.

Response & Feedback Loop

From detection to concrete action

Closing the loop

Alerting & escalation

High‑risk events generate alerts that appear prominently on the dashboard, with full context: system, user, endpoint, parameters, and model decision.

Blocking & policies

Operators can block IP addresses, tune auto‑block rules (for example, risk thresholds for logins), and store those as reusable policies.

Automation

Rules can trigger automatic responses like temporary blocks for brute‑force attempts or extra verification for suspicious transactions.

Continuous improvement

The Dashboard also logs its own actions (alerts, blocks, status), enabling analysis of which rules are too strict or too weak and how models can be improved over time.

In short: Events → Logs → AI classification → Risk & category → Alert → Human or automatic action (block, monitor, investigate).
